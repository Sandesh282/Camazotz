## Phase 1 (Design Disaster)

### Objective
Explain the intent behind your intentionally flawed design.

**Brief Description:** > RecipeX is a recipe discovery and AI cooking assistant app. The intended user flow allows individuals to browse a list of dishes, view detailed step-by-step cooking instructions, and chat with an AI assistant to get culinary help. We have intentionally disrupted this core flow to maximize cognitive friction and violate standard UX heuristics.

---

### Intentional UX Issues Introduced

#### Issue 1: Login Activity
* The screen throws everything at the user at once and calls it a layout.

#### Issue 2: Main Activity
* Key actions are playing hide and seek, and they’re winning.

#### Issue 3: Recipe Detail Screen
* The layout looks like it went through a blender and no one bothered to fix it.

#### Issue 4: Recipe Form Activity
* Filling the form feels less like input and more like a test of patience.

#### Issue 5: AI Chat Activity
* The UI is packed so tightly it feels like it’s trying to conserve screen space for a future that never comes.

#### Issue 6: Item Category Row
* Some items hog all the space while others are just there for emotional support.

#### Issue 7: Top Layout
* The app generously donates its top space to the notification bar, because who needs visible content anyway.

#### Issue 8: Missing Header
* The app refuses to introduce itself, leaving users to guess where they are and why.

### Design Anti-Patterns Used

- **Multiple CTAs:** Every button feels like a gamble, because labels and actions are barely on speaking terms.  
- **Poor visual hierarchy:** The layout has the depth of a brick wall, everything flattened into chaos.  
- **Cognitive overload:** The UI makes users work harder than the app itself.  
- **Inconsistent components:** The design language switches mid-screen like it forgot what it was trying to be.
